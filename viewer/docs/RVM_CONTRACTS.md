# RVM Viewer — Frozen Data Contracts

> Schema version: **rvm-bundle/v1**, **rvm-index/v1**, **rvm-review-tags/v1**, **rvm-saved-view/v1**
> These schemas are frozen. All agents import from `viewer/rvm/` modules — never inline a schema.

---

## 1. Bundle Manifest (`model.bundle.json`)

Schema version: `rvm-bundle/v1`

```json
{
  "schemaVersion": "rvm-bundle/v1",
  "bundleId": "sample-plant-001",
  "source": {
    "format": "RVM",
    "files": [{ "name": "sample.rvm", "sha256": "..." }]
  },
  "converter": {
    "name": "rvmparser",
    "version": "x.y.z",
    "mode": "static-preconverted",
    "warnings": []
  },
  "runtime": {
    "units": "mm",
    "upAxis": "Y",
    "originOffset": [0, 0, 0],
    "scale": 1
  },
  "artifacts": {
    "glb": "sample.glb",
    "index": "sample.index.json",
    "tags": "sample.review.xml"
  },
  "coverage": {
    "attributes": true,
    "tree": true,
    "supports": false,
    "reviewTags": true
  },
  "modelClass": "single-bundle"
}
```

**Rules:**
- `schemaVersion` must be exactly `"rvm-bundle/v1"` — rejected otherwise.
- `artifacts.glb` is required; `index` and `tags` are optional.
- `modelClass: "chunked-bundle"` deferred to Phase 2; rejected by loader.
- `sha256` is preserved as-is; not validated in browser.
- Parser: `parseBundleManifest()` in `viewer/rvm/RvmBundleManifest.js`.

## 2. Model Index (`model.index.json`)

Schema version: `rvm-index/v1`

```json
{
  "schemaVersion": "rvm-index/v1",
  "bundleId": "sample-plant-001",
  "units": "mm",
  "upAxis": "Y",
  "nodes": [
    {
      "sourceObjectId": "RVM:12345",
      "canonicalObjectId": "RVM:12345",
      "renderObjectIds": ["mesh_182"],
      "parentCanonicalObjectId": null,
      "name": "PIPE-A",
      "path": "/AREA1/PIPE-A",
      "kind": "pipe",
      "bbox": { "min": [-1, 0, -1], "max": [1, 5, 1] },
      "attributes": { "NPD": "6in", "SPEC": "CS150" }
    }
  ]
}
```

**Rules:**
- Phase 1: `canonicalObjectId === sourceObjectId` (no remap). This equality is enforced at generation time by `tools/gen-rvm-index.js`.
- `renderObjectIds` maps to Three.js mesh UUIDs or names from the GLB.
- `parentCanonicalObjectId: null` for root nodes.
- `bbox` is optional; `attributes` defaults to `{}`.
- Generated by: `node tools/gen-rvm-index.js <rvmparser-attrs.json> <output.json>`.

## 3. Identity Map (`viewer/rvm/RvmIdentityMap.js`)

Runtime class built from `RvmIndex.nodes` at load time.

```
sourceObjectId  ──→  entry { canonicalObjectId, renderObjectIds[] }
canonicalObjectId ──→ entry
renderObjectId ──→ canonicalObjectId
```

**Phase 1 invariant:** `entry.canonicalObjectId === entry.sourceObjectId`.

API:
```js
const map = RvmIdentityMap.fromNodes(index.nodes);
map.lookupBySource(sourceId)           // → entry | null
map.lookupByCanonical(canonicalId)     // → entry | null
map.canonicalFromRender(renderObjId)   // → canonicalObjectId | null
map.renderIdsFromCanonical(canonicalId) // → string[]
```

## 5. Tag XML (`*.review.xml`)

Schema version: `rvm-review-tags/v1`

```xml
<ReviewTags schemaVersion="rvm-review-tags/v1" bundleId="sample-plant-001">
  <Tag id="TAG-001">
    <CanonicalObjectId>RVM:12345</CanonicalObjectId>
    <SourceObjectId>RVM:12345</SourceObjectId>
    <AnchorType>object</AnchorType>
    <Text>Check interference near elbow</Text>
    <Severity>medium</Severity>
    <ViewStateRef>VIEW-003</ViewStateRef>
    <Status>active</Status>
  </Tag>
</ReviewTags>
```

**Unresolved tag policy:** if `CanonicalObjectId` cannot be resolved at import time, the tag is kept with `<Status>unresolved</Status>` and surfaced in diagnostics. Never silently dropped.

## 10. Bundle Generation Pipeline

```
RMSS.rvm
  └─ rvmparser --output-gltf=sample.glb --output-json=sample-attrs.json
       --output-gltf-attributes=true --output-gltf-center=true
       --output-gltf-rotate-z-to-y=true
  └─ node tools/gen-rvm-index.js sample-attrs.json sample.index.json [bundleId]
  └─ node tools/gen-rvm-bundle.js <dir>/   →  model.bundle.json
```

Binary location for assisted mode: `viewer/converters/bin/rvmparser-windows-bin.exe`
(not in repo — build from `third_party/libtess2` + `third_party/rapidjson`).
